### 2.3 核心指标定义

#### 2.3.1 车间负荷率（日）

```
车间负荷率(日) = (白班最大负荷工时 + 晚班最大负荷工时) / 22.5
```

- **白班最大负荷工时** = 当天同车间白班报工中 F_YJY_loadTimes 的 **最大值**（同班次多台设备取最大，不可叠加）
- **晚班最大负荷工时** = 同车间晚班的最大值
- **额定工时** = 22.5h/天（两班制合计）
- **结果上限** = 100%（超过时截断到 1 = 100%）

> 为什么取最大值？同车间同班次多台设备同时运行，工时不可叠加。取最大值代表该班次的"占用"状态。

#### 2.3.2 车间负荷率（周/月）

```
车间负荷率(周/月) = Σ(每日实际工时) / (工作日天数 × 22.5h)
```

- **分子** = 每天白班最大 + 晚班最大 的累加和
- **分母** = 工作日天数 × 22.5h
- **工作日天数**：
  - 周维度：通过 `countWorkingDaysInWeek(year, week)` 计算该周内的工作日数
  - 月维度：通过 `countWorkingDaysInMonth(year, month)` 计算该月内的非周日非法定假日的天数
  - 年维度：从年初到当前日期的工作日总数
  - **周日自动排除**（`isWorkingDay` 中 `dayOfWeek === 0` 返回 false）
  - **法定节假日排除**（`holiday.js` 中 HOLIDAYS_2026 配置）
  - **当前时段封顶**：如果当前时段（本周/本月），工作日天数按当前日期计算，不等完整周/月

#### 2.3.3 设备负荷率（日）

```
设备负荷率(日) = (白班负荷工时 + 晚班负荷工时) / 22.5
```

- 与车间不同：**直接相加**（同一设备的两个班次工时可以累加）
- 白班负荷工时 = 该设备当天白班 F_YJY_loadTimes 的最大值
- 晚班负荷工时 = 该设备当天晚班 F_YJY_loadTimes 的最大值

#### 2.3.4 产出率

```
产出率 = Σ(实际产量 FQuaQty) / Σ(理论产出: F_FD_AimPerProduct × F_YJY_loadTimes)
```

- **分子**：全部/本车间/本设备所有记录 FQuaQty 的累加和
- **分母**：对应记录的 F_FD_AimPerProduct × F_YJY_loadTimes 的累加和
- 当分母为 0 时，产出率为 0
- 结果上限 100%

#### 2.3.5 OEE

```
OEE = Σ(OEE × 产量) / Σ(产量)   ← 按产量加权平均
```

- 直接使用金蝶 `F_YJY_OEE` 字段值（已经是 0-100 格式的百分比值）
- **按产量加权**：OEE 值乘以该记录对应的 FQuaQty 后累加，再除以总产量
- OEE 不设上限（金蝶数据可能出现 > 100% 的值，属于源头数据问题）
- 仅在有产量的记录上计算 OEE
- OEE 的展示层级：
  - 设备详情页：展示该设备加权平均 OEE 及 OEE 趋势
  - 总览页：设备维度 OEE 按产量加权平均

### 2.4 聚合引擎流程

```
_fetchRecords()
  ├── executeBillQueryAll('SFC_OperationReport', { fieldKeys: 23字段, filter: 已审核 })
  ├── _transformSfcRecords()        // 2D 数组 → 内部记录格式
  ├── _applyMachineInline()          // F_FD_Machine 内码 → 设备名称
  ├── _normalizeRecords()            // 日期截断、班次映射、数值 parseFloat
  ├── 去重                           // 全部字段指纹去重
  └── 工序过滤                       // 仅保留"固体灌装"和"液体灌装"

_aggregate(records, dimension)
  ├── _computeDailyWorkshopMetrics()    // 每日车间级: 负荷率/产出率
  ├── _computeDailyEquipmentMetrics()   // 每日设备级: 负荷率/产出率/OEE
  ├── _aggregateToPeriods()             // 周/月/年: 基于工作日的聚合
  │   (年维度时额外按 month 聚合供趋势图使用)
  └── _computeSummary()                 // 全厂汇总: totalOutput/avgLoadRate/avgOutputRate/avgOEE

API 响应提取 (展平为前端友好格式)
  ├── _extractOverview()                // 总览: 指标 + 9车间卡片 + 趋势
  ├── _extractWorkshops()               // 所有车间列表
  ├── _extractWorkshopDetail()          // 车间详情 + 设备列表 + 趋势
  └── _extractEquipmentDetail()         // 设备详情 + 4趋势 + OEE
```

### 2.5 缓存机制

| 层级 | 位置 | TTL | 策略 |
|------|------|-----|------|
| 聚合引擎缓存 | `aggregationEngine._cache` (Map) | 5分钟 (默认) | key=dimension, 存完整聚合结果 |
| 响应层缓存 | `monitor.js` responseCache (Map) | 1小时 | key=`${dimension}_${endpoint}`, 存展平后的 API 响应 |
| 预热 | 服务启动时 | 后台预热 | day/week/month/year 四维度同时预热，不阻塞启动 |

### 4.3 维度切换

所有页面顶部都有日/周/月/年切换按钮，切换后：
1. `currentDimension` 更新
2. URL query 同步更新（`router.replace({ query: { dimension } })`）
3. 触发 `loadData()` 重新请求 API
4. URL 参数持久化，刷新页面保持维度

年维度特殊处理：
- 趋势图按月展示（后端额外聚合 monthlyWorkshopData/monthlyEquipmentData）
- 指标卡片展示全年汇总

## 五、认证系统

### 5.1 登录流程

```
User → 输入用户名密码 → POST /api/auth/login
  → 服务端查 users.json → 验证匹配 → 签发 JWT (24h有效期)
```

### 5.2 请求鉴权

- axios 请求拦截器自动附加 `Authorization: Bearer <token>`
- 服务端 authMiddleware 验证 JWT，过期或无效返回 401
- 前端响应拦截器捕获 401 → 清除 token → 跳转 /login

## 七、已知问题与待办

### 7.1 OEE 超 100% 问题

- **原因**：金蝶源头数据中 F_YJY_OEE 字段部分记录超过 100%（性能指标 > 100% 属于正常现象），极端值如粉体M机台 2026-04-18 记录达 103,137.5%（由负荷工时 0.01h 导致的计算异常）
- **现状**：聚合引擎不设上限，保持原始值
- **待处理**：需确认是否在聚合时对 OEE 做 100% 截断

### 7.2 年维度平均负荷率问题

- **原因**：`_extractOverview()` 中年维度趋势图按月度填充时 `bucket.workingDays = 1` 硬编码导致负荷率计算异常
- **待处理**：修复为按月计算实际工作天数

### 7.3 设备详情 502 错误

- **原因**：部分设备在年维度下某些 periodKey 可能为 undefined（设备在年中才出现），导致 `localeCompare` 报错
- **待处理**：增加 undefined 保护
